# Launch Posts

## One-line pitch

VibeChat is a local terminal bridge for copy-paste vibe coding with ChatGPT, Claude, and other capable chatbots.

## GitHub post

```text
I built VibeChat, an open-source terminal tool for local vibe coding with chatbots.

Why I made it:
Chatbots are great at planning and reasoning, but the real repo, files, tests, and shell live on your machine. I wanted a clean human-in-the-loop bridge instead of giving a model direct control of the computer.

What it does:
- turns chatbot instructions into one structured local request at a time
- runs reads, edits, searches, patches, and shell checks in the active folder
- streams human-readable progress in a full-screen terminal UI
- copies the full machine-readable response back to the clipboard for the next chat turn

GitHub: https://github.com/angdwww/VibeChat
POC built with it: https://github.com/angdwww/BrowserUse-VibeChatPOC
Feedback welcome.
```

## Reddit

```text
Title: I built VibeChat, a local terminal bridge for copy-paste vibe coding with chatbots

Body:
I wanted a workflow where a chatbot could reason about my codebase without directly controlling my machine.

So I built VibeChat. It is a local terminal tool that accepts one structured request at a time, runs local repo operations like reads, edits, searches, patches, and shell commands, then copies the full response back to the clipboard for the next chat turn.

The main idea is:
- the chatbot handles planning and reasoning
- VibeChat handles local execution
- the human stays in control of every request

It also has a full-screen TUI, resumable sessions, usage tracking, and a chatbot-facing SKILLS.md file so the model knows what it can ask for.

Repo: https://github.com/angdwww/VibeChat
Proof of concept built with it: https://github.com/angdwww/BrowserUse-VibeChatPOC

I would love feedback on the workflow, the request format, and the terminal UX.
```
